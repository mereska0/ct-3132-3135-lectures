# Бин деревья поиска(BST)
*в java - TreeSet, C++ - set*

реализуем операции:
- `add(x)`
- `find(x)`
- `remove(x)`
чтобы у всех была ассимптотика O(H), где H - высота дерева

### Инвариант:
![[Алгосы/img/image-2.png]]
$\forall y \in L: y < x$
$\forall r \in R: r > x$

напишем класс для каждого звена:
```
class Node{
int x; //значение
Node l;//левый сын
Node r;//правый сын
Node p;//(optional) родитель
}
```
- find(x) - начиная с родителя сравниваем элементы с х, и перемещаемся вправо(если x < node) или влево(если x > node)
- add(x) - find(x) и если не нашли то вставляем x
- remove(х) - find(x) и либо
 1. спокойно удаляем
 2. ищем максимум, меняем с x и удаляем
 ___
 само по себе бинарное дерево вообще не свэг, если делать add(1), add(2), ... add(n), то все будет добавляться в одну сторону и получится бамбук
 для этого люди выдумывали всякие деревья с доп. инвариантами:
 - AVL-дерево
 - Красно-черное дерево
 - Splay-дерево
 ___
### AVL-дерево:
пусть h(v) - высота эл-та v
**Инвариант:**
![[Алгосы/img/image-3.png]]
	$|h(L) - h(R)| \le 1$ 
	
для балансировки дерева используется *✨Поворот вправо✨*
