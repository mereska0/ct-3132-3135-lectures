### Алиасинг
Что делать, если мы хотим посмотреть на объект типа А как на объект типа  Б?
```
int32_t a = 0x1234;
char b[4];
char *p = &a
b[0] = p[3];
b[1] = p[1]; // norm
_________________________

float f = 1.0f;
int32_t i;
int32_t *p = &f;
i = *p //huinya polnaya
```
Есть важное правило **[Strict Alias analysis](https://stackoverflow.com/questions/98650/what-is-the-strict-aliasing-rule)**
алиасингом можно кастить **легально** из `const` в `not const`, `unsigned`  в `signed` и из `int` в `char`(исключение из правил такое)

Для всего этого прикола даже есть стандартный метод `memcpy(&p1, &p2, sizeof(i));`

### Встроенные функции:
`"1" + "1" = ?`
интуитивно ожидается просто конкатенация, но на деле тк строка это массив чаров, а массив это указатель на первый элемент, то мы получим хуйню полную

конкатенацию нужно писать самому😭😭😭

```
void concat(const char *p1, const char *p2, char *p)
{
	strcpy(p, p1);
	strcat(p, p2);
}
```
