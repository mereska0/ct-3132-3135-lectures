Будем подключать библиотеку [openCV](https://opencv.org/releases/) версии 3.4 
Весь код для подключения есть [тут](https://skkv-itmo.gitbook.io/c-cpp-cookies/build/build-with-libs)

`-m 64` -ключ чтобы собраться в 64 бита 

**Библиотеки которые подключаются с `< >`** 
- стандартные библиотеки - дают с компилятором 
- системные библиотеки - дают с ОС 
- + Include directories
## Файлы библиотек 
**Статические**  - вкомпиливается в .exe 
	расширения: `.lib .a`
	*Линкольщик вшивает файлы из библиотек прям в экзешник*
	+ независим от внешней среды 
	+ запускаемся из коробки 
	- большой экзешник
	- пересборка (компилить полностью проект заново)
**Динамические** - подгружаем файлы из библиотек динамически 
	расширения: `.dll .so .dylib`
	*ОС связывает библиотеку с экзешником во время запуска программы* 
	+ не нужны пересборка 
	- нужна таскать с собой библиотеку 
	
## базовый Layup библиотек (структура папок)
### include
Там заголовочные файлы 
	.h - c/cpp расширение (читайте документацию чтобы понять)
	.hpp - cpp расширение
для clang gcc используется ключ для указании директории `-I`. Если не укажите include директорию заголовочные файлы отвалятся, а точнее просто ничего не скомпилиться 
**пример команды для сборки:** `clang -m64 src\main.c -I"..\library\opencv\build\include\"`
### Lib/lib64/x64
тут лежат библиотеки чаще всего статические
VC - VS компилятор
**папка lib** 
- name.lib - релизная сборка 
- named.lib - дебаг сборка
**папка bin**
	лежат вспомогательные экзешники + динамические библиотеки 
	`.lib` - библиотека импорта (там только импорты статических библиотек для ОС)
**Команды для сборки**
	`-L` - Library directories
	`-l` - library files 
	**пример команды сборки:** `-L ..\library\opencv\build\x64\vc15\lib\ -lopencv_world3415`
	dll файлы сначала ищется в рабочей директории (**где вы запустились!**), дальше в системных путях 
	*Если вы хотите чтобы всегда искалась в системном пути* 
		добавьте ее в системную переменную (enviroment variable) `PATH`

## Внутренние и внешние функции 
мы хотим чтобы некоторые функции были не видны для пользователя библиотеки поэтому надо настраивать видимость 
- функции с параметром `static` точно не будут видны снаружи
- функции с параметром `export` будут видны снаружи
*Как будет выглядеть export под вашу ОС*

|   ОС    |                               Экспорт                               |
| :-----: | :-----------------------------------------------------------------: |
| Windows |                     по дефолту функцию не видно                     |
|   src   |         `__declspec(dllexport) int my_func(int a) { ... }`          |
|  Unix   |                     по дефлту все функции видны                     |
|   src   | `__attribute__((visability("defoult"))) int my_func(int a) { ... }` |

### Пример использования макросов для кроссплатформенности

```
#ifdef _WIN32
  #define EXPORT __declspec(dllexport)
#else
  #define EXPORT __attribute__((visibility("default")))
#endif

EXPORT int my_func(int a) {
    return a * 2;
}
```

## Сборка статической библиотеки 
1. делаем чистую компиляцию
```
-c .\lib\src\image-viewer.c \              # компилируем исходник (без линковки)
    -I .\lib\include\ \                    # путь к твоим заголовкам (.h)
    -m64 \                                 # 64-битная сборка
    -o image_viewer.obj \                  # выходной объектный файл
    -I ..\library\opencv\build\include\    # путь к заголовкам OpenCV
```

2. из исполняемого файла сделать статическую библиотеку 
	`llvm-lib` - помогает делать статическую библиотеку идет сразу с компилятором 
	`llvm-lib /machine:x64 /out:image_viewer.lib image_viewer.obj`
3. компилируем программу которая использует мою библиотеку
```
# Компилируем main.c и линкуем с ТВОЕЙ библиотекой и OpenCV
gcc main.c \
    image_viewer.lib \                          # твоя библиотека
    -I .\lib\include\ \                         # пути к твоим .h
    -I ..\library\opencv\build\include\ \       # пути к OpenCV .h
    -L ..\library\opencv\lib\ \                 # пути к OpenCV .lib
    -lopencv_core451 \                          # библиотеки OpenCV
    -lopencv_imgcodecs451 \                     # (могут отличаться версии)
    -lopencv_imgproc451 \
    -m64 \
    -o my_program.exe
```
**Не забудьте положить динамические библиотеки рядом с экзешником или добавить их в PATH**
## Сборка динамической библиотеки 
1. делаем из статической библиотеки динамическую 
```
clang -shared -o image_viewer.dll ^           # создаём DLL
    -I ..\library\opencv\build\include\ ^     # путь к заголовкам OpenCV
    -I .\lib\include\ ^                       # путь к твоим заголовкам
    .\lib\src\image_viewer.c ^                # твой исходный код
    -L ..\library\opencv\build\x64\vc15\lib\ ^ # путь к библиотекам OpenCV
    -lopencv_world3415                        # линкуем OpenCV (всё в одном)
```
после этого появится:
```
image_viewer.dll          # САМАЯ ГЛАВНАЯ - динамическая библиотека
image_viewer.lib          # Import library (для линковки с программами)
```

2. компилируем программу которая использует динамическую библиотеки 
```
gcc main.c ^                    # компилируем главный файл программы
    -I .\lib\include\ ^         # путь к твоим заголовкам (image_viewer.h)
    image_viewer.lib ^          # import library (связывает с DLL)
    -o my_program.exe           # имя выходного файла
```
